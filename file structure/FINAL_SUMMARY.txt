💡 KEY INSIGHTS:
================================================================================

TECHNOLOGY STACK:
  • Frontend: Next.js 15 + React 19 + TypeScript + Tailwind CSS
  • Backend: Next.js API Routes
  • Database: Google Sheets API (no SQL)
  • Auth: Simple JWT (hardcoded username/password)
  • Deployment: Vercel
  • State: React hooks + localStorage (draft saving)

RATING SYSTEM:
  • A=5, B=4, C=3, D=2, E=1
  • 15 categories (4 soft skill + 4 hard skill + 5 attitude + 2 ibadah)
  • Ibadah only shown during Ramadan
  • Total Points = Average × 5
  • Predikat = Round(Average) → A/B/C/D/E

PAGES:
  1. Penilaian (Rating) - Manager & Supervisor access
     - Rate employees with expandable cards
     - Draft saving to localStorage
     - Batch submission to Google Sheets
     
  2. Rekapan (Recap) - Manager only (password protected)
     - Group ratings by employee
     - View detail by rater
     - Export CSV/PDF
     
  3. Leaderboard - Public (shareable link)
     - Normalized scores (fairness algorithm)
     - Rankings with ratings
     - Outlet statistics

DATA FAIRNESS:
  Problem: Different outlets have different number of raters
  Solution: Normalize each employee's score based on outlet average
  Formula: normalized = (rawAverage - outletAverage) + globalAverage
  Result: Fair comparison across outlets
